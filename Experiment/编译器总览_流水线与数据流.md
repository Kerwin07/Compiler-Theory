# 编译器整体流水线与数据流详解

## 一句话总结

**输入**一个用类 C 语言编写的源程序文件，经过词法分析→语法分析→语义分析/代码执行，**输出**语义分析报告和程序执行结果。

## 二、逐阶段走查

以下用一段简单的 C 风格演示程序走一遍完整流程：

```c
#include <stdio.h>       ← 被词法器跳过

int main() {
    int x = 5;
    int y = 10;
    int z = x + y;
    printf(z);
    return 0;
}
```

### 阶段 1：词法分析 → Token 序列

词法器读取源程序，按最长匹配策略输出 Token：

```
INT_KW: int      ← 类型关键字
ID: main         ← 函数名
LPAREN: (        ← 左括号
RPAREN: )        ← 右括号
LBRACE: {        ← 函数体开始
INT_KW: int
ID: x
ASSIGN: =
INT: 5
SEMI: ;
INT_KW: int
ID: y
ASSIGN: =
INT: 10
SEMI: ;
INT_KW: int
ID: z
ASSIGN: =
ID: x
PLUS: +
ID: y
SEMI: ;
ID: printf       ← 函数调用名
LPAREN: (
ID: z
RPAREN: )
SEMI: ;
RETURN: return
INT: 0
SEMI: ;
RBRACE: }
```

`#include <stdio.h>` 被 PREPROC 规则跳过，不出现在输出中。

### 阶段 2：语法分析 → AST

LL(1) 预测分析器按文法归约 Token 序列，Collector 栈自底向上构造 AST：

```
S
  INT_KW('int')
  ID('main')
  LPAREN('(')
  RPAREN(')')
  Block
    LBRACE('{')
    StmtList
      Stmt                         ← int x = 5;
        INT_KW('int')
        ID('x')
        DeclRest
          ASSIGN('=')
          Expr → INT(5)
        SEMI(';')
      StmtList
        Stmt                       ← int y = 10;
          INT_KW('int')
          ID('y')
          DeclRest
            ASSIGN('=')
            Expr → INT(10)
          SEMI(';')
        StmtList
          Stmt                     ← int z = x + y;
            INT_KW('int')
            ID('z')
            DeclRest
              ASSIGN('=')
              Expr
                Comp → Term → Power → Factor → ID('x')
                Comp' → PLUS → Term → Power → Factor → ID('y')
            SEMI(';')
          StmtList
            Stmt                   ← printf(z);
              ID('printf')
              StmtTail
                LPAREN('(')
                Args → Expr → ID('z')
              RPAREN(')')
              SEMI(';')
            StmtList
              Stmt                 ← return 0;
                RETURN('return')
                Expr → INT(0)
                SEMI(';')
              StmtList(eps)
    RBRACE('}')
```

如果遇到语法错误，panic recovery 会跳过输入直到同步 Token 并恢复。

### 阶段 3：语义分析 → 报告

遍历 AST，构建带嵌套作用域的符号表，类型推导，表达式常量求值：

```
===== Semantic Analysis Report =====

-- Info (4) --
  defined variable 'x' (type=int, value=5) at Stmt#1
  defined variable 'y' (type=int, value=10) at Stmt#2
  defined variable 'z' (type=int, value=15) at Stmt#3
  RETURN value = 0 [type=int] at Stmt#6

-- Symbol Table --
  x  type=int  value=5   defined=Y  used=Y  definedAt=#1  usedAt=#3
  y  type=int  value=10  defined=Y  used=Y  definedAt=#2  usedAt=#3
  z  type=int  value=15  defined=Y  used=Y  definedAt=#3  usedAt=#5
```

### 阶段 4：代码执行 → 输出

树遍历解释器执行 AST，维护变量环境 `Env`。遇到 `printf` 内置函数时输出到控制台：

```
15          ← printf(z)，z = x + y = 5 + 10 = 15
Program returned: 0
```

## 四、核心算法对照

| 算法 | 实现位置 |
|------|---------|
| 正则→后缀（调度场算法） | `regex_nfa_dfa.cpp` |
| Thompson 构造法（正则→NFA） | `regex_nfa_dfa.cpp` |
| 子集构造法（NFA→DFA） | `regex_nfa_dfa.cpp` |
| Hopcroft DFA 最小化 | `dfa_minimize.cpp` |
| 贪心最长匹配分词 | `lexer_runner.cpp` |
| CFG 加载与解析 | `ll1.cpp` |
| FIRST/FOLLOW 不动点迭代 | `ll1.cpp` |
| LL(1) 预测分析表构建 | `ll1.cpp` |
| 栈驱动 LL(1) 分析 + Collector AST 构造 | `ll_parser.cpp` |
| Panic Recovery 错误恢复 | `ll_parser.cpp` |
| 递归下降 AST 遍历（语义分析） | `semantic.cpp` |
| 嵌套作用域符号表管理 | `semantic.cpp` |
| 树遍历解释器（代码执行） | `codegen.cpp` |
| 内置函数 printf / scanf | `codegen.cpp` |

## 五、如何运行

### 一键运行

```powershell
cd Experiment
.\build_and_run.ps1
```

### 分步运行

```powershell
# 1. 词法分析（查看分词）
cd Lexer
.\lexer_runner.exe

# 2. 语法分析（生成 AST）
cd ..\Parser
.\ll_parser.exe

# 3. 语义分析
cd ..\Semantic
.\semantic_runner.exe

# 4. 代码执行
cd ..\Codegen
.\codegen_runner.exe
```

### scanf 交互式运行

```powershell
cd Codegen
.\codegen_runner.exe          # 手动输入数字，回车
# 或用管道：
echo 42 | .\codegen_runner.exe
```

## 六、修改源程序方法

编辑 `Lexer\source_char.txt`，然后重新运行流水线。支持两种语法风格：

**C 风格**（推荐）：
```c
#include <stdio.h>
int main() {
    int x = 5;
    printf(x);
    return 0;
}
```

**旧风格**（向后兼容）：
```c
x = 5;
y = 10;
return x + y;
```
