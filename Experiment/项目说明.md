# 简易编译器项目说明

## 项目概述

本项目是一个**完整的简易编译器前端 + 树遍历解释器后端**，采用 C++17 编写，运行于 Windows 平台（MSYS2 ucrt64，g++）。

编译器将一种**类 C 语言**的源代码经过词法分析、语法分析、语义分析后，直接由树遍历解释器执行并输出结果。

---

## 源语言简介

源语言是一种语法简化的类 C 语言子集，支持以下特性：

- **数据类型**：`int` 声明，语义分析阶段可自动推断 `float` 和 `bool` 类型
- **变量**：通过 `int x = 5;` 或 `int x;` 声明，或旧风格 `x = 5;` 首次赋值即声明
- **表达式**：支持 `+` `-` `*` `/` `^`（指数）、一元负号（`-a`）、6 种比较运算符（`==` `!=` `>` `<` `>=` `<=`）
- **运算符优先级**（从低到高）：`== != > < >= <=` < `+ -` < `* /` < `^` < 一元 `-`
- **语句**：
  - 类型声明：`int x = 5;` / `int x;`
  - 赋值语句：`x = 5;`
  - 函数调用：`printf(x);` / `scanf();`
  - 条件语句：`if (expr) { ... } [else { ... }]`
  - 循环语句：`while (expr) { ... }`
  - 块语句：`{ ... }`（嵌套作用域）
  - 返回语句：`return expr;`
  - 空语句：`;`
- **函数定义**：`int main() { ... }`

### 示例源程序（`Lexer/source_char.txt`）

```c
#include <stdio.h>

int main() {
  printf("This a test for Compiler");
  int a = 10;
  int b = 3;
  int c = a + b * 2;
  int d = c - 5;
  int e = 2 ^ 4;
  int f = -a;

  if (a == 10) { printf(111); }
  if (a != b)  { printf(222); }
  if (a > b)   { printf(333); }

  int i = 0;
  while (i < 3) { printf(i); i = i + 1; }

  return 0;
}
```

## 系统架构：四大组件

```
源程序(source_char.txt)
        │
        ▼
   ┌─────────┐
   │ 词法分析器 │  →  令牌序列 (source_token.txt)
   └────┬────┘
        │
        ▼
   ┌─────────┐
   │ 语法分析器 │  →  抽象语法树 (ast.json)
   └────┬────┘
        │
   ┌────┴────┐
   ▼         ▼
┌──────┐  ┌───────┐
│语义分析│  │代码执行│
└──┬───┘  └───┬───┘
   ▼         ▼
语义报告    执行结果
```

### 1. 词法分析器（Lexer）

**任务**：将字符流切分为 Token 序列（55 种类型）。

| 程序 | 输入 | 输出 | 算法 |
|------|------|------|------|
| `dfa_builder.exe` | `grammer.txt`（55 条正则规则） | `dfa` | 正则→后缀式→Thompson NFA→子集构造 DFA |
| `dfa_minimizer.exe` | `dfa` | `mindfa` | Hopcroft 算法最小化 |
| `lexer_runner.exe` | `mindfa` + `source_char.txt` | `source_token.txt` | 贪心最长匹配；跳过 WS/注释/预处理指令 |

### 2. 语法分析器（Parser）

**任务**：将 Token 序列按 CFG 文法归约为 AST（49 条产生式）。

| 程序 | 输入 | 输出 | 算法 |
|------|------|------|------|
| `ll_table_builder.exe` | `grammar_cfg.txt` | `ll_table.txt` | FIRST/FOLLOW 不动点迭代→LL(1) 预测分析表 |
| `ll_parser.exe` | `ll_table.txt` + `source_token.txt` | `ast.json` | 栈驱动 LL(1) + Collector 栈 AST 构造 + Panic Recovery |

### 3. 语义分析器（Semantic）

**任务**：遍历 AST 进行语义检查和诊断。

| 程序 | 输入 | 输出 | 分析内容 |
|------|------|------|---------|
| `semantic_runner.exe` | `ast.json` | `semantic_report.txt` | 嵌套作用域符号表、类型推导/检查、表达式求值、use-before-def、未使用变量检测 |

### 4. 代码执行器（Codegen）

**任务**：树遍历解释器直接执行 AST。

| 程序 | 输入 | 输出 | 支持特性 |
|------|------|------|---------|
| `codegen_runner.exe` | `ast.json` | execution_trace.txt + 控制台 | 表达式求值、控制流、printf/scanf、return 传播 |

## 各阶段输入输出对照表

| 阶段 | 程序 | 输入 | 输出 |
|------|------|------|------|
| 词法（离线1） | `dfa_builder.exe` | `grammer.txt` | `dfa` |
| 词法（离线2） | `dfa_minimizer.exe` | `dfa` | `mindfa` |
| **词法（运行时）** | `lexer_runner.exe` | `mindfa` + `source_char.txt` | **`source_token.txt`** |
| 语法（离线） | `ll_table_builder.exe` | `grammar_cfg.txt` | `ll_table.txt` |
| **语法（运行时）** | `ll_parser.exe` | `ll_table.txt` + `source_token.txt` | **`ast.json`** |
| **语义（运行时）** | `semantic_runner.exe` | `ast.json` | **`semantic_report.txt`** |
| **代码执行（运行时）** | `codegen_runner.exe` | `ast.json` | **execution_trace.txt** + 控制台输出 |

## 一键运行

```powershell
cd Experiment
.\build_and_run.ps1
```

脚本自动运行：词法 → 语法 → 语义 → 代码执行四个阶段，输出各阶段结果。
